## Courses API — run tests (no copy-paste)

Start the backend: `cd BE && uvicorn main:app --reload`

From the directory that **contains** `BE/` (e.g. `Source/`):

### Flow: see data first, delete later

1. **CRUD (no delete)** — creates one sample course, lists, gets, updates; rows stay in DB.
2. **Seed** — adds 5 more courses.
3. When done inspecting: **delete all** (admin).

```bash
bash BE/test/run_courses_tests.sh crud        # create + list + get + update only
bash BE/test/run_courses_tests.sh seed        # 5 courses + list
bash BE/test/run_courses_tests.sh delete_all  # removes every course (--yes built in)
```

Or **one shot** (CRUD + seed, still no delete):

```bash
bash BE/test/run_courses_tests.sh all
```

### Python directly

From the same parent directory of `BE/`:

```bash
BE/venv/bin/python BE/test/test_courses_crud.py
BE/venv/bin/python BE/test/test_courses_seed.py
BE/venv/bin/python BE/test/test_courses_delete_all.py --yes
```

From inside `BE/`:

```bash
venv/bin/python test/test_courses_crud.py
```

`delete_all` requires `--yes` when run via Python (safety). The shell wrapper `delete_all` passes `--yes` for you.

### Optional env

- `API_BASE` — default `http://127.0.0.1:8000`
- `LECTURER_UUID` — for seed only

### Files

| File | Purpose |
|------|---------|
| `test_courses_crud.py` | Create → list → get → update (**no delete**) |
| `test_courses_seed.py` | 5 courses + list |
| `test_courses_delete_all.py` | Delete every course (`--yes` required) |
| `run_courses_tests.sh` | `crud` \| `seed` \| `delete_all` \| `all` |

### Table: public.courses

id, created_at, title, description, lecturer_id, schedule, is_complete, course_code, created_by, semester

## Assignments E2E (separate from courses delete_all)

Default run **keeps** course `E2E-ASGN-SCENARIOS` for inspection. Cleanup is explicit:

```bash
bash BE/test/run_assignments_tests.sh              # scenarios only
bash BE/test/run_assignments_tests.sh cleanup      # remove E2E course data only (no scenarios)
bash BE/test/run_assignments_tests.sh teardown     # scenarios, then delete that course/module
```

Python:

```bash
BE/venv/bin/python BE/test/test_assignments_crud.py
BE/venv/bin/python BE/test/test_assignments_crud.py --cleanup-only
BE/venv/bin/python BE/test/test_assignments_crud.py --teardown-after
```

`--cleanup-only` is not part of the scenario test; it only deletes rows for that course code.
